The Meto Platform

From simulation to fabrication

Three integrated tools spanning the full meta-optics development lifecycle — design, manufacture, and validate.

01 — Simulate
MetoSim

Computational Design

FDTD, RCWA, and FEM solvers for electromagnetic wave propagation in nanostructures. Cloud GPUs handle the heavy compute while you work in Python.

02 — Fabricate
MetoFab

Nanofabrication

Structured configs feed directly into fabrication workflows. Seamless handoff from simulation geometry to process-ready specifications.

03 — Validate
MetoLab

Experimental Testing

Close the loop between computation and experiment. Compare simulated and measured optical responses in a unified data environment.

Core Capabilities

Built for nanophotonics research

Everything a researcher needs to go from device concept to publication-quality simulation results.

⚡

GPU-Accelerated FDTD

Full-wave electromagnetic solver running on cloud B200/A100 GPUs. Submit from Python, get results in minutes not hours.

🔬

Material Library

Built-in permittivity data for Si, SiO2, TiO2, Au, Al and more at telecom wavelengths. Extensible with custom materials.

📐

Pydantic Configs

Type-safe simulation configurations validated before submission. Catch errors locally, not after waiting for a GPU.

☁️

Cloud-Native

No local GPU needed. Simulations execute remotely on managed infrastructure. Pay only for compute time.

📊

HDF5 Results

Structured, checksummed output files with full metadata. Reproducible, shareable, and ready for downstream analysis.

🐍

Python-First SDK

Clean, Pythonic API designed for Jupyter workflows. Three lines to submit a simulation; one to visualize.

System Design

Hybrid async architecture

FastAPI gateway with Redis task queue and Modal GPU workers. Independent scaling of API and compute layers.

Client Python SDK (metosim)
│ HTTPS + API Key Auth
Gateway FastAPI REST API PostgreSQL
│ Redis Task Queue
Compute FDTD Engine (GPU) Mesh + Materials
│ Upload Results
Storage S3 Object Store (HDF5)
